# AI & Brain Principles

## Stance

**Anti-chat. Brain-first. Deterministic before AI.**

## Layer rule

```
UI → Services → Brain/Engine → Knowledge → Data → AI (explain/motivate only)
```

## Free tier

No OpenAI. No LLM. No AI inference.

- SmartShop: rule-based baskets, manual offers
- FitnessAI: Fitness Brain rule engines (`ai/config.ts` disabled)
- RecipeAI: deterministic Tonight + fail-closed allergen gates

Do not brand free-tier features "AI" to consumers.

## Premium (future)

Only tier for real AI: offer search, basket optimisation, consumption prediction, cross-app integration, decision support.

## AI may (future)

Explain reasoning · motivate · educate · personalise tone · offline tag enrichment (non-blocking)

## AI must never

Calculate calories/macros · select workouts/meals · generate recommendations · override brain · bypass allergen gates

## Brain-first build order

1. Evidence and formulas  
2. Engine with explainable output (`ruleId`)  
3. Thin service  
4. Minimal UI last  

## Explainability

Default: outcome only. Tap "Why?": plain steps. Never: confidence scores, hypothesis labels, formula walls.

## Safety

- Allergens: EU-14, pre-LLM, fail-closed (RecipeAI)
- Health: documented formulas only (FitnessAI)
- Copy: calm, honest, no sparkles-as-AI

Fitness Brain full spec: `apps/FitnessAI/docs/FITNESS_BRAIN.md`
