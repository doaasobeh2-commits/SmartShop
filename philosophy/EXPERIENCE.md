# Experience Model

How the ecosystem should *feel* — the 2030 quality bar.

## The shift

| Avoid (2026 patterns) | Target (2030) |
|----------------------|---------------|
| Three apps, three homes | One household pulse; apps are lenses |
| Engine branding ("Fitness Brain", "SmartShop AI") | Intelligence is ambient |
| Forms and completeness scores | Confidence from behavior |
| Coach / Analytics tabs | "Why?" on tap, inline |
| User administers weekly plans | "Tonight is handled" |
| Macro grids | One meaningful number per domain |

## Per-lens moments

**SmartShop:** Open → one card: *where to shop, how many items, why now.*

**RecipeAI:** Zero onboarding if household exists → one dinner suggestion → cook → one-tap shop.

**FitnessAI:** Open → *calories left · today's session · recovery* — three tokens, one screen.

## Hidden vs visible

| Show | Hide |
|------|------|
| Today's action, calories remaining, one offer | Hypotheses, confidence, formulas |
| "Running low" (≤3 items) | Full inventory |
| One-sentence meal reason | KRS breakdown |
| Have/need on recipe (tap) | Pantry manager |

**Correction required:** one-tap when inference is wrong (*"Still have rice?"*) — not yet built.

## Future: Household Home

Shop + dinner + body cards on one surface. SmartShop natural shell. Independent repos remain.
